Key insight: Today, associates are stuck in Stage 3 and 4 — manually enriching and orchestrating. The product's job is to automate enrichment (AI reads documents and matches against questions) and provide orchestration (a dashboard that shows status at a glance). Stages 1, 2, and 5 are human decisions — the product supports them, it doesn't replace them.

The associate's pain: They are the human glue across all three dimensions. They read a document from the depth layer, figure out which vertical question it answers, update the horizontal workstream status, then chase the next missing piece. All in Excel. All manually. The product automates the connections between these three dimensions.

Current State

Manual, fragmented, error-prone
1
Deal enters diligence — someone creates a copy of the Excel template and emails it to the deal team
2
Associate opens Box — manually browses folders looking for relevant documents uploaded by advisors and counterparties
3
Reads each document — scans through PDFs and reports to find data points that answer gating items
4
Updates the spreadsheet — manually types findings into the tracker, updates statuses, adds notes
5
Emails the latest version — sends updated tracker to the team. Version control via filename dates
6
Chases missing info — manually tracks what's outstanding, sends follow-up emails, no automated reminders
7
Principal asks for status — associate has to manually compile a summary. No real-time visibility
→

Work Package State

Automated, connected, real-time
1
Deal enters diligence — system auto-creates work package with templated questions and workstreams based on deal type
2
Box is connected — system monitors the deal's Box folder via dedicated email address. New docs are automatically ingested
3
AI reads documents — extracts relevant findings and matches them against outstanding gating items and questions
4
Dashboard updates — findings surface with source links and confidence scores. Associate reviews and confirms ("golden answer")
5
Single source of truth — everyone sees the same live dashboard. No version control issues. Collaboration built in
6
Automated orchestration — system flags outstanding items, shows what's stale, surfaces what changed since last login
7
Principal opens dashboard — sees AI-generated summary, progress by workstream, and can drill into any gating item

The transferable pattern: Every client has the same structural problem — a work package that needs to be enriched from multiple sources by multiple people under time pressure, where the current process is manual, fragmented, and version-controlled by email. Conquer's expertise is in recognising this pattern and building the tooling to automate the enrichment and orchestration layers. Apollo is the PE investment version of a universal workflow.

reframed as
Work Package Framework

Instantiate & Track

Auto-create work package on deal entry. Template questions and workstreams. Track lifecycle progress. Show completeness.

Enrich & Connect

AI reads Box documents, extracts findings, matches to outstanding questions. Surfaces golden answers with provenance. Automates the associate's reading.

Orchestrate & Decide

Show what's answered, what's stale, who's responsible. Daily/weekly AI summaries. Surface when the work package is decision-ready.

Why the reframe matters: The Excel tabs describe what exists today. The modules describe what we're building. The work package framework describes why it matters — and gives everyone (designer, developer, client) a shared mental model for every design and architecture decision. Every feature should answer: does this help the work package get from entry to decision faster?
